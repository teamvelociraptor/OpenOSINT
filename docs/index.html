Why OpenOSINT?

The AI decides which tools to run and in what order.
You just provide the target.

⚡

Native Tool Use

Built on Anthropic's tool use API — no brittle JSON ReAct loops. The model autonomously chains tools and pivots on findings.

◈

10 OSINT Modules

Email validation, username enumeration (17 platforms), domain WHOIS + DNS, IP geolocation, phone analysis, breach checks, metadata extraction, and Google dork generation.

◉

Beautiful Terminal UI

Rich-powered interface with live tool execution, spinners, structured reports, and auto-saved Markdown files — designed to produce beautiful screenshots.

⊕

Multi-Provider

Works with Anthropic (default), OpenAI, or any Ollama local model. Switch with one environment variable.

⌕

Zero Friction Setup

One API key to get started. Optional integrations for HaveIBeenPwned and AbuseIPDB unlock deeper breach and reputation data.

☏

Interactive + One-shot

Conversational REPL for iterative investigations, or a single openosint investigate <target> command for scripting and pipelines.

Installation

01

Clone & install

git clone https://github.com/openosint/openosint
cd openosint
bash setup.sh          # creates venv, installs deps
source .venv/bin/activate
02

Set your API key

export ANTHROPIC_API_KEY=sk-ant-...
# or add to .env file (copy from .env.example)

Get a key at console.anthropic.com. Free tier available.

03

Run your first investigation

openosint investigate john@example.com
# or launch the interactive REPL
openosint

Or install via pip

pip install openosint

AI Providers

Swap providers with a single environment variable.

Default

Anthropic

Best results. Uses Claude's native tool use API with stop_reason: tool_use — no prompt engineering needed.

export ANTHROPIC_API_KEY=sk-ant-...
export OPENOSINT_MODEL=claude-sonnet-4-20250514

OpenAI

Compatible with GPT-4o and later models via the OpenAI function calling API.

export OPENAI_API_KEY=sk-...
export OPENOSINT_PROVIDER=openai

Ollama

Run locally with any Ollama model that supports function calling (llama3.1, qwen2.5, etc.). No API costs.

export OPENOSINT_PROVIDER=ollama
export OPENOSINT_MODEL=llama3.1
export OLLAMA_BASE_URL=http://localhost:11434

CLI Reference

Interactive Mode

openosint

Launches the interactive REPL. Type any target to start an investigation. The agent maintains conversation context across messages.

openosint ❯ john.doe@gmail.com
openosint ❯ What about related domains?
openosint ❯ save
openosint ❯ quit

One-Shot Investigation

openosint investigate <target> [--save] [--output FILE]

Ideal for scripts and pipelines. Outputs the report to stdout and optionally saves to a Markdown file.

openosint investigate 8.8.8.8
openosint investigate example.com --save
openosint investigate user@corp.com --output report.md

Configuration

openosint config --show
openosint config --provider anthropic
openosint config --model claude-opus-4-5

Settings are stored in ~/.config/openosint/config.json. Environment variables always override saved settings.

Python API

from openosint.config import Config
from openosint.display import Display
from openosint.agent import OpenOSINTAgent

config = Config.load()
display = Display(quiet=True)
agent = OpenOSINTAgent(config, display)

report = agent.investigate("example.com")
print(report)

Responsible Use

✓ Intended for

  • Security research and penetration testing (with authorization)
  • Investigative journalism on matters of public interest
  • Digital forensics and incident response

  • Verifying your own digital footprint

✗ Not for

  • Stalking, harassment, or doxing
  • Unauthorized surveillance of individuals
  • Activities violating GDPR, CCPA, or applicable privacy law
  • Automated mass harvesting of personal data

OpenOSINT only queries publicly available data sources. Users are solely responsible for ensuring their use complies with applicable laws and regulations.
